Intermediate Git Commands with Examples

> git config
// Configure the author name and email address to be used with your commits. Note that Git strips some characters from user.name. Using the '--global' tag makes this the defaut for all commits and repositories.

git config --global user.name "CodingEasy" git config --global user.email CodingEasy@example.com

> git add
// Add one or more files to staging (index): This command gets the modifed/new files ready to be committed.

git add <filename> git add *

> git commit
// Commit staged changes. This command commits the changes in the staging area to the repository. This is the final step before pushing the changes to the remote repository.

git commit -m "Commit message"

> git push
// Send changes to the master branch of your remote repository: Here 'origin' is the remote repository. and 'master' is the branch that will be pushed to.

git push origin branch_name

> git pull
// Pull changes from a remote repository. Used incase there were changes in the source repo while you were working on the repository.

git pull

> git branch
// When you want to work on new feature, but don't want to commit it to the main branch. Make a new branch and start working on it..

git branch -d {branch-name}
